About

Architecture & deployment guide

API Endpoints

  • GET /health — liveness check, model status
  • GET /historical — last 90 days training data
  • POST /predict — forecast N days ahead
    Body: {"days": 30}

MLOps Pipeline

  • GitHub Actions cron job at 2:00 AM IST daily
  • Ingest → Train (Prophet) → Drift detect (Evidently)
  • Metrics logged to DagsHub MLflow
  • Model pushed to Hugging Face Hub as model.pkl

Deploy FastAPI (Render)

  1. Push code to GitHub
  2. New Web Service on render.com
  3. Set env var HF_REPO_ID
  4. Render auto-builds the Dockerfile
  5. Paste the Render URL into the input above

Model

Facebook Prophet with multiplicative seasonality, trained on 5 years of NSE closing prices. Default ticker: TCS.NS. Change via the TICKER GitHub Actions variable.
